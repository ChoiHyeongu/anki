# 📘 Flashcard SRS App – 기능 명세서 (MVP)

## 0. 문서 목적

본 문서는 **Anki 스타일 SRS 기반 플래시카드 앱**의
👉 **MVP 기능 범위와 동작을 UI 디자인 기준으로 명확히 정의**하여
기획–개발 간 해석 차이를 없애는 것을 목표로 한다.

---

## 1. 핵심 기획 원칙

- 카드 단위 학습
- 사용자 자기 평가 기반 SRS
- 신규 카드 제한 / 복습 카드 무제한
- 덱은 **DB에 사전 정의된 것만 사용 (읽기 전용)**
- 차별화 기능 제외, Anki 정석을 최대한 따름

---

## 2. 화면별 기능 명세

---

# 2-1. 메인 대시보드 (Deck List / 라이브러리)

### 📍 화면 목적

- 오늘 처리해야 할 학습량을 **덱 단위로 직관적으로 보여줌**
- 학습 진입의 기준점 역할

---

### 📌 덱 리스트 아이템 구성

각 덱은 다음 정보를 포함한다:

#### 1️⃣ 덱 이름

- 예: `TOEIC Essential 2000`

#### 2️⃣ 학습 지표 숫자 3개 (오른쪽 정렬)

**순서 고정 (좌 → 우)**

| 색상        | 의미               | 설명                                          |
| ----------- | ------------------ | --------------------------------------------- |
| 파란색/흰색 | 신규 (New)         | 오늘 처음 학습하게 될 카드 수                 |
| 주황/빨간색 | 학습 중 (Learning) | 오늘 배웠지만 아직 단기 반복이 필요한 카드 수 |
| 초록색      | 복습 (Due)         | 오늘 복습 타이밍이 도래한 카드 수             |

📌 예시
`12   5   0`
→ 복습 12 / 학습 중 5 / 신규 0

---

### 📌 숫자 의미 규칙

- 해당 숫자들은 **“오늘 이 덱에서 처리해야 할 학습량”**만 나타냄
- 숫자가 **모두 0이 되면**
  - 해당 덱의 오늘 학습 목표 달성 상태

---

### 📌 신규 카드 제한 (Anki 기본값)

- 신규 카드: **하루 최대 20개**
- 덱별 또는 전역 설정 가능 (MVP에서는 전역 권장)
- 복습(Due) / 학습 중(Learning)은 **제한 없음**

---

### 📌 덱 선택 동작

- 덱 탭 → 해당 덱 기준으로 학습 세션 시작
- 오늘 처리할 카드가 0개인 경우:
  - “오늘 학습 완료” 상태 안내

---

## 2-2. 학습 화면 – Front 상태

### 📍 화면 목적

- 카드의 질문(Front)을 보여줌
- 사용자의 기억 회상을 유도

---

### 📌 구성 요소

- 카드 Front 텍스트
- 상단:
  - 세션 진행도: `현재 / 오늘 목표`
    - 예: `12 / 50`

- 하단 안내:
  - `Tap to reveal answer`

---

### 📌 동작

- 화면 탭 → Back 상태로 전환

---

## 2-3. 학습 화면 – Back + 평가 상태

### 📍 화면 목적

- 정답 확인
- 사용자의 자기 평가를 통해 SRS 상태 확정

---

### 📌 구성 요소

1️⃣ 카드 정보

- 단어
- 발음
- 의미
- 예문
- 동의어

2️⃣ 카드 상태 정보 (하단)

- Reviews (누적 복습 횟수)
- Interval (다음 복습까지의 일수)
- Ease (%)

> ※ 학습 판단을 방해하지 않도록 시각적 강조는 최소화

---

### 📌 평가 버튼 (고정)

| 버튼   | 의미            |
| ------ | --------------- |
| 다시   | 기억 못함       |
| 어려움 | 불완전하게 기억 |
| 알맞음 | 적절히 기억     |
| 쉬움   | 매우 잘 기억    |

---

### 📌 평가 동작

- 버튼 선택 시:
  - 카드 SRS 상태 즉시 반영
  - 다음 카드로 자동 이동

---

## 2-4. Undo 기능

### 📍 목적

- 사용자의 실수로 인한 SRS 왜곡 방지

---

### 📌 Undo 정의 (MVP 기준)

- Undo는 **“마지막 평가 1건”만 되돌릴 수 있음**
- Undo 동작 시:
  - 직전에 평가한 카드로 복귀
  - 해당 카드의 SRS 상태를 **평가 전 상태로 완전 복구**

- Undo 가능 타이밍:
  - 다음 카드가 표시된 상태에서만 가능

- 다음 카드를 평가하면:
  - 이전 카드에 대한 Undo는 불가능

---

## 2-5. 세션 종료 화면 (완료 화면)

### 📍 화면 목적

- 오늘 학습 종료를 명확히 선언
- 오늘 학습 결과 요약
- 다음 행동 안내

---

### 📌 구성 요소

#### 1️⃣ 완료 메시지

```
오늘의 학습 완료!
```

---

#### 2️⃣ 오늘 학습 요약

```
복습 카드: 38
신규 카드: 12
```

---

#### 3️⃣ 다음 복습 안내

```
다음 복습은 내일이에요
```

또는

```
내일 복습 예정 카드: 42
```

---

#### 4️⃣ 버튼

- `[홈으로 이동]` (Primary)
- 뒤로 가기 / 스와이프 뒤로가기 비활성화

---

## 2-6. 카드 추가 화면

### 📍 목적

- 신규 카드 입력

---

### 📌 필수 요소

- 덱 선택 (드롭다운)
  - DB에 존재하는 덱만 표시

- Front 입력
- Back 입력
- 저장 버튼

---

## 3. 빈 상태 (Empty State) 정의

### 1️⃣ 덱에 카드가 없는 경우

```
이 덱에는 아직 카드가 없습니다
카드를 추가해 주세요
```

### 2️⃣ 오늘 학습할 카드가 없는 경우

```
오늘의 학습을 모두 완료했어요
```

---

## 4. 명시적으로 제외한 기능 (MVP 범위 밖)

- 로그인 / 계정
- 클라우드 동기화
- 스트릭 / XP / 랭킹
- 통계 대시보드
- 카드 공유
- 자동 정답 판별
- AI 카드 생성

---

## 5. MVP 성공 기준

- 사용자가 매일 “오늘 할 것”을 헷갈리지 않는다
- 실수(오탭)로 SRS가 망가지지 않는다
- 학습을 시작하고, 끝내는 지점이 명확하다
